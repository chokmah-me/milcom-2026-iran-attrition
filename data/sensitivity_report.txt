==============================================================================
SENSITIVITY FINDINGS
==============================================================================

1. EXPIRY WINDOW IMPACT ON H2 (Pre-Programmed)
   The pre-auth expiry window is the MOST SENSITIVE parameter for H2.
   Tight windows (15-20 days) cause H2 to collapse ~day 18-22.
   Deep windows (45-60 days) allow H2 to sustain fire as long as H1.
   The discriminating power between H1 and H2 depends critically on this.

2. EXPIRY WINDOW IMPACT ON H1 (Active C2)
   Zero effect. H1 cells receive live targeting regardless of pre-auth.
   H1 decline is driven entirely by launcher attrition + missile depletion.
   This is the CONTROL: H1 curves should be identical across all windows.

3. EXPIRY WINDOW IMPACT ON H3 (Mixed)
   Moderate effect. Connected cells (H1-like) are unaffected.
   Disconnected cells (H2-like) hit the expiry cliff.
   The window size determines how visible the H2 component is in the mix.

4. REAL-WORLD DISCRIMINATION
   We are now at day 23 of the real conflict. Key observables:
   - If launch cadence shows a SECOND sharp decline around days 25-40
     (beyond the launcher-attrition decline already observed), this supports
     H2 with a ~25-40 day planning horizon.
   - If cadence continues smooth attrition-driven decline, H1 is more likely.
   - If cadence holds STEADY at a low level (indicating rationing), this is
     consistent with H1 (adaptive command choosing to conserve) but not H2
     (which cannot choose to conserve, only fire or go stale).

5. CALIBRATION AGAINST REAL DATA
   Day 1: 480 BMs (real) vs model ~170 (model undercounts, expected:
   model uses MRBMs only, real includes SRBMs and cruise missiles).
   Day 9: ~40 BMs (real) vs model ~15-25 (reasonable for MRBM-only).
   Day 12: ~4 missiles/wave (real) vs model 8-15/day (close).
   The shape of the decline curve matches well: steep front-loading,
   plateau at low level, slow grind. This validates the attrition model.
